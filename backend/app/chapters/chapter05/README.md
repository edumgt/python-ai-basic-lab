# CHAPTER05 - 선형회귀

- 주제: 선형회귀
- 목표: 입력 특성과 목표값의 선형 관계를 학습하고 MSE로 오차를 해석합니다.
- 실행: `python practice.py`

---

## 핵심 개념

- **회귀 문제**: 정답이 숫자일 때 사용하는 가장 기본적인 예측 모델입니다.
- **기울기와 절편**: 입력이 1만큼 바뀔 때 예측값이 얼마나 움직이는지 설명합니다.
- **MSE(평균제곱오차)**: 예측값과 실제값의 차이를 제곱해 평균낸 값으로, 낮을수록 좋습니다.
- **최소제곱법**: 오차를 제곱해 더한 값이 가장 작아지도록 직선을 찾는 대표적인 선형회귀 학습 아이디어입니다.

## 먼저 구분하면 쉬워요

| 개념 | 쉬운 뜻 | 예시 |
|------|------|------|
| **연속값** | 숫자가 자연스럽게 이어지는 값 | 종가, 거래량, 수익률 |
| **이산값** | 답이 몇 개로 끊어져 있는 값 | 상승/하락, 강세/약세 |
| **회귀** | 연속값을 예측하는 문제 | "내일 종가가 얼마인가?" |
| **분류** | 이산값을 고르는 문제 | "내일 오를까, 내릴까?" |

이 챕터는 그중에서도 **회귀**, 즉 **연속값 예측**에 집중합니다.

## 최소제곱법을 아주 쉽게 보면

선형회귀는 점들 사이에 직선을 하나 긋고,  
그 직선이 실제 데이터와 **얼마나 멀리 떨어졌는지(오차)** 를 봅니다.

최소제곱법은 그 오차들을 그냥 더하지 않고 **제곱해서 더한 뒤**,  
그 값이 가장 작아지는 직선을 찾는 방법입니다.

왜 제곱하냐면:

- `+오차`와 `-오차`가 서로 상쇄되지 않게 하려고
- 많이 틀린 점에 더 큰 벌점을 주려고

그래서 초보자 감각으로는  
**"전체 틀린 정도의 벌점을 가장 작게 만드는 직선을 찾는다"** 고 이해하면 충분합니다.

## 언제 쓰나요?

- 매출·가격·수요처럼 **연속형 숫자**를 예측할 때
- 특성 변화가 결과에 **직선 형태로 반영되는지** 빠르게 확인할 때
- 복잡한 모델을 쓰기 전에 **기준선(baseline)** 모델이 필요할 때

## 결과 해석 체크리스트

- 실행 결과의 `mse`가 너무 크지 않은지 확인합니다.
- 기본 실행에서 `64.143399...`처럼 같은 값이 반복되면, 이는 고정된 `random_state=42`와 동일한 연습 데이터 생성 설정을 사용하기 때문이라는 점을 이해합니다.
- 특성 수가 늘어날수록 오차가 줄어드는지 비교합니다.
- 설명 탭에서 회귀와 분류의 차이를 먼저 정리하고 실습을 봅니다.

## 웹앱 실습 순서

1. 챕터를 선택한 뒤 **설명 탭**에서 회귀 문제의 목표를 읽습니다.
2. 상단 **실행** 버튼을 눌러 `mse` 값을 확인합니다.
3. **웹앱 실습 탭**에서 `샘플 수`, `특성 수`, `노이즈`, `테스트 비율`, `랜덤 시드`를 바꿔 다시 실행합니다.
4. 결과 탭에서 왜 값이 고정되었는지 설명 카드와 바뀐 `mse`를 함께 읽습니다.
5. 차트 탭에서 다른 회귀형 결과와 비교해봅니다.
6. 다음 챕터인 `chapter06`과 이어서 분류 문제와 차이를 정리합니다.
