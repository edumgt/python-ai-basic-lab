# CHAPTER06 - 로지스틱 회귀

- 주제: 로지스틱 회귀
- 목표: 주가 상승/하락 이진 분류에서 확률 출력과 결정 경계를 해석합니다.
- 실행: `python practice.py`

---

## 핵심 개념

- **분류 문제**: 결과가 숫자가 아니라 `상승/하락`, `강세/약세`처럼 범주일 때 사용합니다.
- **시그모이드 함수**: 점수를 0~1 확률로 바꿔서 해석을 쉽게 만듭니다.
- **결정 경계**: 확률이 0.5를 넘는지 기준으로 클래스를 나눕니다.

## 먼저 구분하면 쉬워요

| 개념 | 쉬운 뜻 | 예시 |
|------|------|------|
| **이산값** | 답이 몇 개의 칸으로 끊어진 값 | 상승/하락, 강세/약세/횡보 |
| **분류** | 이산값 중 하나를 고르는 문제 | "내일 오를까, 내릴까?" |
| **이진 분류** | 정답이 2개인 분류 | 상승/하락, 스팸/정상 |
| **다중 클래스 분류** | 정답이 3개 이상인 분류 | 하락/보합/상승 |

이 챕터의 로지스틱 회귀 예시는 **이진 분류**입니다.  
즉, `0 또는 1`, `하락 또는 상승`처럼 두 답 중 하나를 맞히는 연습입니다.

## 언제 쓰나요?

- 다음 날 상승/하락처럼 두 가지 상태 중 하나를 판단하는 **기초 분류 문제**
- 모델이 어떤 특성 때문에 예측했는지 **해석 가능성**이 중요할 때
- 복잡한 모델 전에 **기본 성능 기준선**을 만들고 싶을 때

## 결과 해석 체크리스트

- 실행 결과의 `accuracy`가 0.5보다 충분히 높은지 확인합니다.
- 같은 데이터라도 임계값을 바꾸면 정밀도·재현율이 달라진다는 점을 기억합니다.
- 설명 탭에서 확률 해석과 선형회귀와의 차이를 함께 정리합니다.

## 웹앱 실습 순서

1. 챕터 실행으로 `accuracy`를 확인합니다.
2. 결과 탭에서 확률 기반 분류라는 점을 메모합니다.
3. **주식 AI 실험실**로 이동해 로지스틱 회귀를 선택하고 같은 분류 흐름을 웹에서 다시 실습합니다.
4. 실험실에서 정확도·AUC·매수 정밀도를 함께 비교합니다.
