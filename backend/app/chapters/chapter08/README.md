# CHAPTER08 - 랜덤포레스트

- 주제: 랜덤포레스트
- 목표: 여러 트리의 투표로 다음 날 주가 방향 예측 성능을 높이고 F1을 해석합니다.
- 실행: `python practice.py`

---

## 핵심 개념

- **앙상블**: 여러 모델을 묶어 하나보다 안정적인 결과를 만듭니다.
- **배깅(Bagging)**: 데이터와 특성을 조금씩 다르게 뽑아 여러 트리를 학습합니다.
- **F1 점수**: 정밀도와 재현율의 균형을 보는 대표 지표입니다.

## 언제 쓰나요?

- 복잡한 비선형 패턴을 빠르게 잡고 싶을 때
- 설명력과 성능의 균형이 필요한 실무형 분류 문제에서
- 단일 트리보다 **안정적인 일반화 성능**이 필요할 때

## 결과 해석 체크리스트

- 실행 결과의 `f1`이 로지스틱 회귀보다 개선되는지 비교합니다.
- 랜덤포레스트는 특성 중요도를 함께 제공해 해석에 도움을 줍니다.
- 데이터가 적으면 과도한 트리 수보다 검증 전략이 더 중요할 수 있습니다.

## 웹앱 실습 순서

1. 챕터 실행으로 `f1` 값을 확인합니다.
2. `chapter07`와 번갈아 실행하며 단일 트리 대비 변화를 관찰합니다.
3. **주식 AI 실험실**에서 랜덤포레스트를 선택해 정확도·AUC·수익률을 다시 비교합니다.
4. 차트 탭에서 feature importance와 포트폴리오 곡선을 함께 해석합니다.
