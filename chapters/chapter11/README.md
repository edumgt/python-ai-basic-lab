# CHAPTER11 - 검증 전략

- 주제: 검증 전략
- 목표: train/valid/test 분리와 교차검증의 차이를 이해합니다.
- 실행: `python practice.py`

---

## 핵심 개념

- **데이터 분할**: 학습·검증·테스트 역할을 분리해야 일반화 성능을 볼 수 있습니다.
- **교차검증**: 여러 번 나눠 평가해서 우연한 편차를 줄입니다.
- **시계열 검증**: 미래 정보를 과거에 섞지 않도록 순서를 지켜야 합니다.

## 언제 쓰나요?

- 데이터가 적어 한 번의 분할만으로 판단하기 어려울 때
- 하이퍼파라미터를 비교하면서 과적합을 막고 싶을 때
- 금융처럼 시간 순서가 중요한 문제를 다룰 때

## 결과 해석 체크리스트

- 실행 결과의 `cv_mean`을 보고 단일 정확도보다 안정적인지 확인합니다.
- 랜덤 셔플 검증과 시계열 검증은 결과가 크게 다를 수 있습니다.
- 좋은 모델보다 **좋은 검증 절차**가 먼저라는 점을 기억합니다.

## 웹앱 실습 순서

1. 챕터 실행으로 교차검증 평균값을 확인합니다.
2. `chapter10` 지표와 연결해 어떤 점수가 검증에서 안정적인지 비교합니다.
3. 주식 관련 웹앱에서 모델을 바꾸며 결과 편차가 큰지 확인합니다.
4. 미니 프로젝트(`chapter112`) 전에 반드시 검증 기준을 정리합니다.
