# 시계열 모델 2: PatchTST, TFT, iTransformer

> Transformer를 시계열에 더 잘 맞게 바꾸면 어떤 점이 좋아질까요?

---

## 이 문서의 역할

- `04.md`는 최신 시계열 Transformer 계열 모델을 소개하는 문서입니다.
- PatchTST, TFT, iTransformer가 각각 어떤 문제를 잘 푸는지 정리합니다.
- 이 문서부터는 "모델 구조를 모두 구현"하는 것보다 **어떤 입력 구조와 해석 포인트가 중요한지** 이해하는 데 초점을 둡니다.

## 🌟 초등학생도 이해할 수 있어요

Transformer를 시계열에 더 잘 맞게 바꿨다는 말은,
쉽게 말해 **주가 공부를 더 잘하는 새로운 공부법**을 만든 것과 비슷합니다.

- PatchTST: 긴 문제집을 **몇 장씩 나눠서** 읽기
- TFT: 가격, 거래량, 계절 같은 **여러 힌트를 같이 보기**
- iTransformer: 여러 종목이 **서로 어떻게 연결되는지** 보기

## 최신 모델 이름도 낱말로 나누면 훨씬 쉽습니다

| 용어 | 영어 낱말 | 말 그대로 뜻 | 이렇게 기억하면 쉬워요 |
|------|------|------|------|
| PatchTST | `patch` + `TST` | 조각 + 시계열 Transformer | 긴 차트를 조각내 보는 모델 |
| patch | 조각 | 작은 덩어리 | 긴 데이터를 몇 칸씩 묶은 조각 |
| Temporal | 시간의 | 시간과 관련된 | 날짜 흐름과 관련 있다는 뜻 |
| Fusion | 합치기, 융합 | 여러 것을 섞기 | 가격, 거래량, 계절 힌트를 함께 보기 |
| TFT | Temporal Fusion Transformer | 시간 정보를 섞어 보는 변환기 | 여러 힌트를 같이 보는 시계열 모델 |
| iTransformer | 모델 이름의 한 부분 `i` + Transformer | Transformer 계열 이름 | 이 문서에서는 `여러 변수 관계를 잘 보는 Transformer 계열` 정도로 기억하면 충분합니다. |

## 왜 일반 Transformer만으로는 부족할까요?

시계열 데이터는 언어와 조금 다릅니다.

- 긴 길이의 연속 구간이 많고
- 여러 변수와 여러 종목이 동시에 움직이고
- 계절성과 이벤트가 섞여 있고
- 실무에서는 해석 가능성도 중요합니다

그래서 Transformer를 시계열에 맞게 바꾼 모델들이 등장했습니다.

## 여기서 자주 나오는 단어 풀이

| 단어 | 쉬운 뜻 | 왜 필요한가 |
|------|------|------|
| 다변량 | 한 종류가 아니라 여러 종류 값을 같이 봄 | 가격만이 아니라 거래량, 계절, 금리까지 함께 볼 수 있습니다. |
| 정적 정보 | 시간에 따라 자주 안 바뀌는 정보 | 회사 이름, 업종 같은 고정 힌트입니다. |
| 미래에 아는 정보 | 예측 시점 전에 이미 알 수 있는 정보 | 요일, 달, 휴일 일정 같은 값입니다. |
| 해석 가능성 | 모델이 왜 그렇게 판단했는지 설명할 수 있는 정도 | 실무에선 점수만큼 설명도 중요합니다. |
| 변수축 | 시간보다 "무엇을 측정했는가" 쪽 기준 | 여러 특성 사이 관계를 볼 때 자주 쓰는 말입니다. |
| 패치(patch) | 긴 데이터를 잘게 자른 묶음 | 너무 긴 시계열을 효율적으로 보기 위한 방법입니다. |

### 한자도 함께 보면 덜 낯섭니다

- `다변량(多變量)`은 **여러(多) 개의 값(量)** 을 함께 본다는 뜻입니다.
- `정적 정보(靜的 情報)`는 **조용하게 잘 안 바뀌는 정보**라고 생각하면 쉽습니다.
- `해석 가능성`은 결과를 **사람 말로 풀어 설명할 수 있는 힘**입니다.

## 1. PatchTST

PatchTST는 긴 구간을 작은 조각(`patch`)으로 나누어 봅니다.

| 관점 | 설명 |
|------|------|
| 핵심 생각 | 긴 시계열을 짧은 구간 묶음으로 처리 |
| 장점 | 긴 입력을 더 효율적으로 다룸 |
| 잘 맞는 상황 | 장기간 흐름, 패턴 반복, 긴 히스토리 |

### 이렇게 이해하면 쉽습니다

- 긴 차트를 하루씩 전부 읽는 대신
- `8일 묶음`, `16일 묶음`처럼 잘라 본다고 생각하면 됩니다

예를 들어 64일 차트를 볼 때:

- 1~8일
- 9~16일
- 17~24일

처럼 작은 조각으로 나누면,  
아주 긴 차트도 덜 복잡하게 볼 수 있습니다.

### patch라는 말이 낯설다면

긴 만화책을 한 페이지씩 읽는 대신  
`1~8페이지`, `9~16페이지`처럼 묶어서 보는 것과 비슷합니다.

그러면 모델은 "하루하루"보다 "짧은 구간의 패턴"을 더 잘 볼 수 있습니다.

---

## 2. TFT

TFT(Temporal Fusion Transformer)는 여러 종류의 정보를 함께 다루는 데 강합니다.

예를 들어:

- 가격 시계열
- 거래량
- 달/분기/계절
- 회사 고정 정보

를 한 모델 안에서 다루며,
어떤 입력이 중요한지도 비교적 잘 보여줍니다.

> 쉬운 비유: 담임 선생님이 시험 점수만 보는 게 아니라, 숙제, 출석, 발표, 계절 행사까지 함께 보는 것

| 관점 | 설명 |
|------|------|
| 핵심 생각 | 정적 정보 + 시계열 정보 + 미래에 아는 정보까지 함께 사용 |
| 장점 | 해석 가능성, 다변량 입력 |
| 잘 맞는 상황 | 매출, 수요, 예약률, 여러 설명변수가 있는 예측 |

### 설명변수라는 말도 자주 나옵니다

`설명변수`는 결과를 설명하는 데 쓰는 입력 힌트입니다.

예를 들어 호텔 예약률 예측이라면:

- 어제 예약률
- 주말 여부
- 계절
- 이벤트 여부

같은 값들이 모두 설명변수입니다.

---

## 3. iTransformer

iTransformer는 시계열의 "시간축"보다 "변수축" 관계를 더 적극적으로 다루는 쪽으로 유명합니다.

| 관점 | 설명 |
|------|------|
| 핵심 생각 | 여러 변수 간 관계를 더 효율적으로 학습 |
| 장점 | 다변량 시계열에서 강점 |
| 잘 맞는 상황 | 여러 종목, 여러 지표, 센서 다변량 데이터 |

> 쉬운 비유: 한 명의 성적만 보는 게 아니라, 반 친구들이 서로 같이 잘하는 과목이 있는지 같이 보는 것

### 변수축 관계란 무슨 뜻일까요?

예를 들어 같은 날에

- 거래량이 커지고
- 변동성이 커지고
- 다른 관련 종목도 같이 움직였다면

시간 흐름만 본 것이 아니라  
**여러 변수끼리 서로 어떤 관계로 묶여 움직였는지** 보는 것입니다.

## 세 모델 비교

| 모델 | 강한 포인트 | 추천 상황 |
|------|------|------|
| PatchTST | 긴 구간을 조각내 처리 | 긴 히스토리 예측 |
| TFT | 다양한 입력과 해석 가능성 | 수요예측, 멀티피처 |
| iTransformer | 변수 간 관계 학습 | 다변량 시계열 |
