# 모델링 기본기: 튜닝, 검증, 평가, 군집화

> 좋은 모델은 우연히 맞히는 모델이 아니라, 다시 돌려도 믿을 수 있는 모델입니다.

---

## 이 문서의 역할

- `02.md`는 모델링의 공통 기본기를 모아 둔 문서입니다.
- 하이퍼파라미터 튜닝, 교차 검증, 성능 평가, 군집화 해석을 한 번에 정리합니다.
- 모델 종류보다 **"모델을 어떻게 믿을 수 있게 만들 것인가"**에 초점을 둡니다.

---

모델링 기본기는 쉽게 말해 **시험공부를 똑똑하게 하는 방법**입니다.

- 하이퍼파라미터 튜닝: 연필, 지우개, 공부 시간처럼 **공부 방법을 조절**
- 교차 검증: 시험을 한 번만 보는 게 아니라 **여러 번 확인**
- 성능 평가: "몇 문제 맞았지?"를 점검
- 군집화 해석: 반 친구들을 **비슷한 특징끼리 모둠 짜기**

즉, 좋은 모델은 그냥 우연히 한 번 맞힌 모델이 아니라  
**다시 시험 봐도 비슷하게 잘하는 모델**입니다.

## 이 문서의 어려운 말도 먼저 뜯어볼게요

| 용어 | 한자/영어 | 말 그대로 뜻 | 쉬운 이해 |
|------|------|------|------|
| 검증 | 檢證 / *validation* | 검사해서 증명함 | 진짜 실력인지 중간 점검 |
| 평가 | 評價 / *evaluation* | 값을 매겨 봄 | 몇 점짜리 모델인지 보기 |
| 교차 | 交叉 / *cross* | 서로 엇갈리게 바꿈 | 시험 묶음을 번갈아 바꿔 가며 확인 |
| 하이퍼 | *hyper* | 더 높은, 더 바깥의 | 모델 바깥에서 사람이 정하는 설정 |
| 파라미터 | *parameter* | 기준이 되는 값 | 모델을 움직이는 숫자 설정 |
| 군집화 | 群集化 / *clustering* | 무리로 묶기 | 비슷한 친구들끼리 모둠 만들기 |
| 누출 | 漏出 / *leakage* | 새어 나옴 | 보면 안 될 미래 정보가 새어 들어온 상태 |

## 1. 하이퍼파라미터는 무엇인가요?

하이퍼파라미터는 모델이 배우기 전에 사람이 정하는 설정값입니다.

> 쉬운 비유: 로봇 선수에게 "몇 번 연습할까?", "얼마나 천천히 배울까?", "질문을 얼마나 깊게 할까?"를 정해주는 것과 비슷합니다.

| 모델 | 자주 조정하는 값 | 뜻 |
|------|------|------|
| 로지스틱 회귀 | `C` | 규제를 얼마나 약하게 둘지 |
| 랜덤 포레스트 | `n_estimators`, `max_depth` | 트리 개수, 깊이 |
| GBM | `n_estimators`, `learning_rate`, `max_depth` | 작은 모델 수, 학습 속도, 깊이 |
| 신경망 | 층 수, 뉴런 수, `max_iter` | 구조와 학습 횟수 |
| K-Means | `n_clusters` | 몇 개 무리로 나눌지 |

### 기억할 점

- 하이퍼파라미터는 **정답이 하나가 아닙니다.**
- 데이터가 달라지면 좋은 값도 달라집니다.
- 그래서 반드시 **검증 절차**와 함께 봐야 합니다.

## 자주 헷갈리는 단어를 먼저 정리해볼게요

| 단어 | 쉬운 뜻 | 헷갈리기 쉬운 점 |
|------|------|------|
| 파라미터 | 모델이 학습하면서 스스로 정하는 값 | 가중치처럼 데이터에서 배워집니다. |
| 하이퍼파라미터 | 사람이 먼저 정하는 설정값 | 학습 전에 정하는 버튼값에 가깝습니다. |
| 훈련 데이터(train) | 모델이 공부하는 문제집 | 여기만 잘 맞히면 아직 안심할 수 없습니다. |
| 검증 데이터(validation) | 중간 점검용 문제집 | 튜닝할 때 기준이 됩니다. |
| 테스트 데이터(test) | 마지막 실전 시험지 | 마지막 확인용이라 자주 들여다보면 안 됩니다. |
| 셔플(shuffle) | 데이터를 섞는 것 | 일반 표형 데이터에선 자주 쓰지만, 시계열에서는 조심해야 합니다. |
| 누출(leakage) | 미래 정보가 몰래 들어간 상태 | 실제보다 점수가 비정상적으로 높아질 수 있습니다. |
| 폴드(fold) | 교차 검증에서 나눈 한 조각 | 여러 조각으로 바꿔가며 시험 본다고 생각하면 쉽습니다. |

### 영어를 직역해 보면 더 쉽습니다

- `validation`은 "맞는지 확인"이라는 뜻이라서 **검증**이라고 부릅니다.
- `evaluation`은 "값을 매겨 보기"라서 **평가**입니다.
- `leakage`는 원래 물이 새는 것처럼 **정보가 샌다**는 뜻입니다.
- `cross validation`은 시험 조각을 **엇갈리게(cross)** 바꿔 가며 확인한다는 뜻입니다.

### 아주 쉬운 예시

반 학생 100명 성적표가 있다고 가정해봅시다.

- `훈련 데이터`는 AI가 공부하는 80명
- `검증 데이터`는 공부 방법을 바꿔 보며 확인하는 10명
- `테스트 데이터`는 마지막에 실력을 보는 10명

이때 테스트 학생 답을 미리 본다면 그게 바로 **누출**입니다.

## 2. 왜 교차 검증이 필요할까요?

한 번만 `train/test`로 나누면 운이 좋았는지 나빴는지 알기 어렵습니다.

> 쉬운 비유: 수학 시험을 딱 1장만 풀어 보고 "난 수학 잘해!"라고 말하면 위험하죠.  
> 문제집을 여러 장 풀어 봐야 진짜 실력이 보입니다.

교차 검증은 데이터를 여러 번 나누어 보면서

- 점수가 들쑥날쑥한지
- 평균적으로 괜찮은지
- 특정 구간에서만 잘 맞는지

를 확인하는 방법입니다.

| 방법 | 언제 유용한가 | 주의할 점 |
|------|------|------|
| 단순 train/test split | 빠르게 기준선 볼 때 | 우연 영향 큼 |
| K-Fold | 일반 표형 데이터 | 시계열엔 시간 누출 주의 |
| 시계열 분할 | 과거→미래 흐름 유지 | 셔플 금지 |
| Walk-forward | 실전 투자 흐름과 비슷 | 계산량 증가 |

### K-Fold를 더 쉽게 설명하면

예를 들어 데이터가 5묶음이면:

1. 첫 번째 묶음을 시험으로 쓰고 나머지 4묶음으로 공부합니다.
2. 두 번째 묶음을 시험으로 쓰고 다시 점검합니다.
3. 이 과정을 5번 반복합니다.

그러면 "운 좋게 한 번만 잘 맞은 것인지"를 덜 걱정할 수 있습니다.

## 3. 성능 평가는 무엇을 같이 볼까요?

| 지표 | 의미 | 언제 중요할까 |
|------|------|------|
| `accuracy` | 전체 중 맞힌 비율 | 가장 기본 |
| `precision` | 오른다고 했을 때 실제로 오른 비율 | 매수 신호 품질 |
| `recall` | 실제 상승 중 잡아낸 비율 | 기회를 놓치지 않기 |
| `AUC` | 상승/하락을 얼마나 잘 구분하는지 | 분류 전반 품질 |
| `confusion matrix` | 어떤 실수를 했는지 | FP/FN 구분 |
| 수익률 곡선 | 실제 전략으로 벌었는지 | 퀀트 실전 관점 |

### 퀀트에서는 특히

- 정확도만 높아도 돈을 못 벌 수 있습니다.
- `precision`, `AUC`, 전략 수익률을 같이 보는 습관이 중요합니다.

## 지표 이름을 더 쉽게 읽는 법

| 지표 | 아주 쉬운 말 | 이렇게 읽어보세요 |
|------|------|------|
| accuracy | 전체 시험 점수 | 100문제 중 몇 문제를 맞혔는지 |
| precision | "오른다"라고 말했을 때의 정확함 | 매수 버튼을 눌렀을 때 얼마나 자주 맞았는지 |
| recall | 실제 상승을 얼마나 놓치지 않았는지 | 좋은 기회를 얼마나 챙겼는지 |
| AUC | 점수 순서를 얼마나 잘 매겼는지 | 상승 후보를 전반적으로 잘 앞에 세우는지 |
| confusion matrix | 틀린 방식까지 적어 놓은 표 | 어떤 실수가 많은지 자세히 볼 수 있습니다. |

### 영어 이름이 왜 이런지

- `precision`은 **정밀함**이라는 뜻이라서, "오른다"라고 말했을 때 얼마나 정확히 맞췄는지와 잘 어울립니다.
- `recall`은 원래 **다시 불러낸다**는 느낌이 있어서, 실제 상승 기회를 얼마나 잘 찾아냈는지와 연결됩니다.
- `confusion matrix`는 **헷갈린 결과를 표로 적은 것**이라고 생각하면 쉽습니다.

### FP, FN도 꼭 알아두면 좋아요

- `FP(False Positive)`는 오를 거라고 했는데 실제로는 안 오른 경우입니다.
- `FN(False Negative)`는 안 오른다고 했는데 실제로는 오른 경우입니다.

퀀트에서는 FP가 많으면 쓸데없는 매수가 늘 수 있고,  
FN이 많으면 좋은 기회를 자주 놓칠 수 있습니다.

## 4. 군집화는 왜 여기에 같이 묶였나요?

군집화는 지도학습은 아니지만,
모델링 기본기를 익힐 때 매우 중요합니다.

이유는:

- 데이터 구조를 먼저 이해하게 해주고
- 비슷한 종목 무리를 찾게 해주고
- 이후 포트폴리오, 섹터 해석, 이상치 탐색으로 확장되기 때문입니다.

### 군집화 결과를 볼 때 체크할 것

- 그룹 수를 왜 그렇게 정했는가
- 그룹별 평균 특성이 어떻게 다른가
- 같은 군집 종목이 실제로도 비슷한지
- 군집 결과를 투자 판단이 아니라 **해석의 출발점**으로 쓰고 있는가

### 작은 예시

예를 들어 6개 종목이 있다고 할 때:

- 삼성전자, SK하이닉스, DB하이텍이 한 그룹
- 은행주 2개가 또 한 그룹
- 바이오 1개가 혼자 따로

나왔다면, 컴퓨터는  
"얘네 움직임이 비슷해 보여!"라고 묶은 것입니다.  
이건 정답 발표가 아니라 **시장 지도를 그려 주는 힌트**예요.

## 군집화 용어도 같이 알아두기

| 단어 | 쉬운 뜻 | 기억법 |
|------|------|------|
| 군집(cluster) | 비슷한 것끼리 모인 무리 | 색깔 비슷한 색연필 묶음 |
| 중심점(centroid) | 그 무리의 대표 위치 | 반 평균 키처럼 대표값 |
| K값 | 몇 무리로 나눌지 정한 숫자 | 2팀, 3팀, 4팀처럼 팀 수 |
| 거리(distance) | 서로 얼마나 비슷한지 재는 기준 | 가까우면 비슷, 멀면 다름 |

군집화는 "정답 찾기"보다  
"데이터 안에 어떤 무리가 숨어 있는지 보기"에 더 가깝습니다.

용어가 더 낯설면 [voca.md](voca.md)에서 한자와 영어를 함께 다시 볼 수 있습니다.

## 6. 실수하기 쉬운 포인트

- 검증 없이 하이퍼파라미터만 바꾸면 운 좋은 점수를 진짜 실력으로 착각하기 쉽습니다.
- 시계열인데 데이터를 무작위로 섞으면 미래 정보를 미리 본 셈이 될 수 있습니다.
- 군집화 결과에 이름을 붙일 때는 숫자 근거 없이 "성장주 그룹" 같은 말을 너무 빨리 붙이면 위험합니다.
- 정확도 하나만 보고 모델을 고르면 매수 신호 품질이 떨어질 수 있습니다.
