# Day 2. 모델 기본기: 점수는 왜 여러 개를 볼까?

> 오늘은 "잘 맞는 모델"보다 "믿을 수 있는 모델"이 더 중요하다는 걸 배우는 날입니다.

---

## 오늘의 목표

- `튜닝`, `검증`, `평가`, `군집화`가 무엇인지 아주 쉽게 이해합니다.
- 점수 하나만 보고 모델을 고르면 왜 위험한지 압니다.
- 웹앱에서 같은 모델을 다시 돌려 보며 숫자 읽는 연습을 시작합니다.

---

## 오늘 먼저 기억할 말

좋은 모델은 시험 1번만 잘 본 친구가 아닙니다.  
**다시 풀어도 비슷하게 잘하는 친구**입니다.

그래서 우리는 아래 4가지를 같이 봅니다.

| 이름 | 쉬운 뜻 | 비유 |
|---|---|
| 튜닝 | 설정값 조절하기 | 연필, 지우개, 공부 시간 바꾸기 |
| 검증 | 중간 점검하기 | 모의고사 여러 번 보기 |
| 평가 | 몇 점인지 보기 | 시험 점수표 확인 |
| 군집화 | 비슷한 것끼리 묶기 | 친구들 모둠 만들기 |

---

## 오늘의 핵심 낱말

| 낱말 | 쉬운 뜻 |
|---|---|
| 하이퍼파라미터 | 사람이 먼저 정하는 버튼값 |
| 교차 검증 | 시험 문제를 여러 묶음으로 바꿔가며 확인하는 방법 |
| 정확도 | 전체 중 몇 개를 맞혔는지 |
| 누출 | 보면 안 될 미래 정보가 몰래 들어온 상태 |

---

## 왜 점수 하나만 보면 안 될까요?

예를 들어 어떤 모델이 `정확도 90%`라고 해도,

- 어려운 문제는 놓치고 있을 수 있고
- 특정 구간에서만 우연히 잘했을 수 있고
- 미래 정보를 몰래 본 결과일 수도 있습니다

그래서 아래처럼 여러 질문을 해야 합니다.

- 정말 여러 번 해도 괜찮은가?
- 오를 거라고 했을 때 진짜 잘 맞히는가?
- 시간 순서를 어기지 않았는가?

## 초등학생식 해석 팁

- `accuracy`: 전체 맞힌 개수 보기
- `precision`: "맞다!"라고 말했을 때 진짜 잘 맞았는지 보기
- `AUC`: 전체적으로 잘 구분하는지 보기
- `검증`: 운이 아니라 실력인지 확인하기
