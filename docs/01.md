퀀트 AI 모델은 어려운 이름이 많지만, 아주 쉽게 보면 이렇습니다.

먼저 `모델`이라는 말부터 풀어보면 더 쉽습니다.

- 장난감 자동차는 진짜 자동차를 **작게 본뜬 모형**입니다.
- 지구본은 진짜 지구를 **간단히 옮겨 놓은 모형**입니다.

AI에서 말하는 `모델(model)`도 비슷합니다.

- 현실 세계를 그대로 복사한 것은 아니고
- 중요한 규칙만 골라서
- 컴퓨터 안에 **작은 계산용 모형**으로 옮겨 놓은 것입니다.

그래서 `모델`이라는 표현을 씁니다.  
즉, 모델은 **현실을 흉내 내어 예측하게 만든 계산 모형**입니다.

- **회귀**: "내일 주가가 몇 원쯤 될까?"처럼 숫자를 맞히는 문제
- **분류**: "오를까, 내릴까?"처럼 이름표를 고르는 문제
- **군집화**: "비슷한 종목끼리 같은 무리일까?"를 찾는 문제
- **시계열 모델**: "어제, 오늘, 지난주 흐름까지 같이 보자!" 하는 문제

쉬운 비유로 보면:

- 회귀는 **온도 예보**
- 분류는 **우산 가져갈지 말지 결정**
- 군집화는 **색연필 색깔별로 정리**
- 시계열 모델은 **만화책 줄거리를 앞에서부터 이어 읽기**

라고 생각하면 됩니다.

## 이름이 어려우면 말뿌리부터 보면 쉬워집니다

| 용어 | 한자/영어 | 말 그대로 뜻 | 초등학생식 이해 |
|------|------|------|------|
| 모델 | 模型 / *model* | 본뜬 틀, 모형 | 현실을 간단히 흉내 내는 계산 도구 |
| 회귀 | 回歸 / *regression* | 다시 돌아감 | 숫자가 어디쯤 갈지 맞히는 문제 |
| 분류 | 分類 / *classification* | 종류를 나눔 | `상승`, `하락` 이름표 고르기 |
| 시계열 | 時系列 / *time series* | 시간 순서 줄세우기 | 어제, 오늘, 내일 흐름 이어 보기 |
| 선형 | 線形 / *linear* | 선 모양 | 직선처럼 단순하게 생각하는 모델 |
| 로지스틱 | *logistic* | 점수를 0~1로 눌러 주는 함수 이름 | 확률처럼 보이게 만드는 계산 |
| 전체 주목 | 全體 注目 / *attention*을 쉽게 풀어 쓴 말 | 전체를 보고 눈길 주기 | 전체 노트를 펴 놓고 중요한 줄에 형광펜 칠하기 |

### 왜 이렇게 읽으면 도움이 될까요?

- `모델`은 거창한 기계 이름이 아니라, **현실을 간단히 흉내 내는 계산 틀**이라는 뜻입니다.
- `회귀`를 처음 보면 한자 때문에 어렵지만, 실제로는 **숫자 예측**입니다.
- `로지스틱 회귀`도 이름은 길지만, 이 repo에서는 **상승 확률을 계산하는 기본 분류 모델**로 보면 됩니다.
- `전체 주목`은 낯선 전문 용어를 쉽게 바꾼 표현이라서, **중요한 곳을 골라 보는 능력**으로 이해하면 됩니다.

## 한눈에 보는 모델 분류

| 묶음 | 대표 모델 | 주로 푸는 문제 | 잘하는 일 | 주의할 점 |
|------|------|------|------|------|
| **선형 계열** | 선형회귀, 로지스틱 회귀 | 수치 예측, 상승/하락 분류 | 빠르고 해석 쉬움 | 복잡한 비선형 패턴에는 약함 |
| **경계 기반** | SVM | 분류 | 클래스 사이 여유 공간을 잘 찾음 | 데이터가 크면 느려질 수 있음 |
| **트리 계열** | 결정 트리, 랜덤 포레스트 | 분류, 회귀 | 규칙이 직관적이고 해석 가능 | 단일 트리는 과적합 위험 |
| **앙상블 계열** | Gradient Boosting, XGBoost, LightGBM | 분류, 회귀 | 성능이 강하고 실전 활용도 높음 | 튜닝에 따라 성능 차이 큼 |
| **기본 딥러닝** | MLP | 분류, 회귀 | 복잡한 조합 패턴 학습 | 입력 설계가 중요 |
| **패턴 인식** | CNN | 구간 패턴 탐지 | 국소 패턴, 모양, 필터 학습 | 긴 시계열 문맥은 제한적 |
| **순서 기억** | RNN, LSTM | 시계열 예측 | 시간 순서와 기억을 다룸 | 긴 의존성에서 약하거나 느릴 수 있음 |
| **전체 주목** | Transformer | 시계열, 멀티시계열 | 긴 구간을 한꺼번에 보고 중요 시점 선택 | 데이터와 연산량이 큼 |

## 단어를 쉬운 말로 풀어보기

| 단어 | 쉬운 뜻 | 왜 중요한가 |
|------|------|------|
| 회귀 | 숫자를 맞히는 예측 | 주가, 수익률, 거래량처럼 숫자를 바로 맞힐 때 필요합니다. |
| 분류 | 이름표를 고르는 예측 | `상승`, `하락`처럼 둘 중 하나를 맞히는 문제에서 자주 씁니다. |
| 특성(feature) | 모델이 참고하는 힌트 | 가격, 거래량, 이동평균선 같은 입력값이 모두 특성입니다. |
| 비선형 | 직선으로 설명되지 않는 복잡한 관계 | 시장은 단순 직선보다 구부러진 관계가 많아 트리나 신경망이 필요해집니다. |
| 과적합 | 연습문제만 외우고 새 문제에 약한 상태 | 훈련 데이터는 잘 맞지만 실제 데이터에 약해질 수 있어 꼭 경계해야 합니다. |
| 앙상블 | 여러 모델 의견을 함께 쓰는 방법 | 한 명의 의견보다 여러 명의 의견을 모으면 더 안정적일 때가 많습니다. |
| Attention | 여러 시점 중 중요한 곳에 더 집중하는 계산 | 긴 시계열에서 "언제가 중요했는지" 찾는 데 큰 도움을 줍니다. |
| 해석 가능성 | 모델이 왜 그렇게 예측했는지 설명할 수 있는 정도 | 퀀트에서는 점수뿐 아니라 이유를 설명하는 능력도 중요합니다. |

### 영어 이름도 같이 보면 덜 무섭습니다

- `Random Forest`는 **무작위(Random) + 숲(Forest)** 입니다.
  많은 트리를 숲처럼 모아 놓았다고 생각하면 됩니다.
- `Gradient Boosting`은 **기울기(gradient)를 따라 조금씩 고치며 끌어올리는(boosting)** 방식입니다.
- `Transformer`는 **입력을 다른 표현으로 바꾸는(transform)** 데 강한 모델이라는 뜻입니다.
- `Feature`는 원래 **특징, 특징적인 모습**이라는 뜻이라서 입력 힌트를 가리킬 때 잘 어울립니다.

### 예시로 다시 보면

- `거래량이 갑자기 늘었다`는 것은 **특성 하나가 크게 움직인 것**입니다.
- `랜덤 포레스트가 중요 특성 1위로 거래량을 뽑았다`는 것은 **모델이 그 힌트를 많이 참고했다**는 뜻입니다.
- `훈련 점수는 높은데 테스트 점수가 낮다`면 **과적합을 의심**해야 합니다.

## 왜 AI는 확률로 말할까요?

많은 초보자가 여기서 한 번 막힙니다.

"AI면 그냥 맞다, 틀리다를 딱 말해주면 되지 왜 73%, 41%처럼 확률로 말하지?"

이유는 현실 데이터가 **항상 딱 잘리지 않기 때문**입니다.

### 쉬운 비유

내일 비가 올지 생각해 봅시다.

- 하늘이 흐리고
- 바람이 차고
- 구름이 많으면

"무조건 비가 와!"라고 말하기보다  
"비가 올 가능성이 높아 보여"라고 말하는 게 더 자연스럽습니다.

주가도 똑같습니다.

- 거래량이 늘고
- 최근 5일 수익률이 좋고
- 이동평균선 위에 있어도

반드시 오른다고 보장할 수는 없습니다.  
그래서 AI는 **정답을 외치는 기계**라기보다  
**가능성을 계산하는 기계**에 더 가깝습니다.

### 조금 더 정확히 말하면

AI 모델은 입력을 보고 이렇게 생각합니다.

- 상승 쪽 증거가 얼마나 많은지
- 하락 쪽 증거가 얼마나 많은지
- 어느 쪽이 더 그럴듯한지

를 숫자로 계산합니다.

그 결과가 `0.82`라면

- "상승이라고 생각한다"
- 그런데 그 자신감은 **82% 정도다**

라고 읽을 수 있습니다.

### 왜 이게 더 좋을까요?

확률로 말하면 좋은 점이 많습니다.

1. **자신감의 크기**를 알 수 있습니다.
   `51%` 상승과 `92%` 상승은 같은 상승 예측이지만 느낌이 전혀 다릅니다.
2. **기준선을 정할 수 있습니다.**
   예를 들어 `55%` 이상일 때만 매수 신호를 내게 할 수 있습니다.
3. **위험 관리를 할 수 있습니다.**
   확률이 애매하면 쉬어 가고, 확률이 높을 때만 행동하는 전략을 만들 수 있습니다.

### 그래서 로지스틱 회귀가 자주 먼저 나오나요?

맞습니다.  
로지스틱 회귀는 점수를 **0~1 사이 확률처럼** 바꿔 주기 때문에

- "오를까?"
- "얼마나 자신 있지?"

를 함께 보기 좋습니다.

그래서 이 repo에서도 로지스틱 회귀를 **분류의 첫 기준선 모델**로 자주 사용합니다.

### 한 줄로 기억하면

AI가 확률로 말하는 이유는  
**현실이 100% 확실한 문제보다, 가능성을 비교해야 하는 문제에 더 가깝기 때문**입니다.

## 퀀트 문제를 모델에 연결하면

| 퀀트 질문 | 추천 모델 묶음 | 이유 |
|------|------|------|
| 내일 오를까, 내릴까? | 로지스틱 회귀, SVM, RF, GBM | 이진 분류에 적합 |
| 다음 주 수익률은 몇 %일까? | 선형회귀, 트리 회귀, 딥러닝 회귀 | 숫자 예측 문제 |
| 어떤 특성이 제일 중요한가? | 로지스틱 회귀, RF, GBM | 중요도와 계수 해석이 쉬움 |
| 같은 움직임의 종목끼리 묶을 수 있을까? | K-Means 등 군집화 | 정답 없이 구조 탐색 |
| 긴 흐름 속 중요한 구간을 찾고 싶다 | LSTM, Transformer | 시계열 문맥 반영 |
| 여러 종목과 계절성까지 함께 보려면? | Transformer 계열, TFT, iTransformer | 다변량 구조에 강함 |

### 작은 예시

예를 들어 삼성전자 차트를 보고 이런 질문을 할 수 있어요.

- "내일 가격이 **61,500원쯤**일까?" → 회귀
- "내일 **상승 / 하락** 중 무엇일까?" → 분류
- "삼성전자랑 SK하이닉스는 **비슷한 무리**일까?" → 군집화
- "최근 30일 흐름을 다 보고, **어느 날이 중요했는지** 찾을까?" → 시계열 모델

이 repo에서는 웹앱에서 바로 볼 수 있도록 **MLP와 뉴런 계산 시각화**를 잘 준비해 두었습니다.

### 3. 시계열 확장 모델

- RNN
- LSTM
- Transformer

이들은 개념 이해 비중이 크며, 웹앱에서는 **직접 같은 모델을 모두 구현하기보다**
입력 구조, 예측 흐름, 특성 해석을 중심으로 연습하게 됩니다.

## 모델 이름이 왜 이렇게 다른가요?

- `로지스틱 회귀`는 이름에 회귀가 들어가지만 이 repo에서는 주로 **분류 모델**로 사용합니다.
- `랜덤 포레스트`는 많은 **결정 트리**를 모아 투표시키는 방식입니다.
- `그래디언트 부스팅`은 앞 모델이 틀린 부분을 다음 모델이 조금씩 고쳐 가는 방식입니다.
- `MLP`는 여러 층의 기본 신경망을 뜻합니다.
- `Transformer`는 순서 데이터를 볼 때 중요한 위치를 넓게 비교하는 데 강합니다.

이름이 어렵더라도, 처음에는  
`선형 계열은 빠르고 단순`, `트리 계열은 규칙 찾기`, `딥러닝은 복잡한 패턴 찾기`  
정도로 나누어 기억하면 훨씬 편합니다.

용어가 더 헷갈릴 때는 [voca.md](voca.md)를 함께 보면
한자, 영어, 쉬운 뜻을 한 번에 다시 확인할 수 있습니다.
