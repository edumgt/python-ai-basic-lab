## 1. 의사결정나무 (Decision Tree)

### 원리

트리는 각 노드에서 **불순도(impurity)**를 가장 많이 줄이는 특성·임계값을 선택해 데이터를 분기합니다.

$$\text{Gini} = 1 - \sum_{k} p_k^2 \qquad \text{Entropy} = -\sum_k p_k \log_2 p_k$$

$$\text{Information Gain} = I(\text{parent}) - \frac{n_L}{n}I(\text{left}) - \frac{n_R}{n}I(\text{right})$$

```
루트 노드: 모멘텀_20 > 0.5?
├── Yes → RSI > 0.3?
│         ├── Yes → 매수 (Leaf, Gini=0.12)
│         └── No  → 횡보 (Leaf, Gini=0.41)
└── No  → 변동성 < -0.5?
          ├── Yes → 매도 (Leaf, Gini=0.08)
          └── No  → 횡보 (Leaf, Gini=0.45)
```

### 과적합과 가지치기 (Pruning)

```python
import numpy as np
import pandas as pd
from sklearn.tree import DecisionTreeClassifier, export_text
from sklearn.metrics import roc_auc_score
from sklearn.preprocessing import StandardScaler

np.random.seed(42)
n = 600

# 팩터 데이터 생성
X = np.column_stack([
    np.random.randn(n),   # 모멘텀
    np.random.randn(n),   # RSI
    np.random.randn(n),   # 변동성
    np.random.randn(n),   # 거래량 변화
])
y = (0.4 * X[:, 0] + 0.2 * X[:, 1] - 0.15 * X[:, 2] + np.random.randn(n) * 0.6 > 0).astype(int)

split = int(n * 0.8)
X_tr, X_te = X[:split], X[split:]
y_tr, y_te = y[:split], y[split:]

# 깊이별 트레이드오프 확인
depths = [2, 3, 5, 8, 10, None]
print(f"{'max_depth':<12} {'train AUC':<12} {'test AUC':<12} {'leaf 수'}")
print('-' * 50)
for d in depths:
    dt = DecisionTreeClassifier(max_depth=d, random_state=42)
    dt.fit(X_tr, y_tr)
    tr_auc = roc_auc_score(y_tr, dt.predict_proba(X_tr)[:, 1])
    te_auc = roc_auc_score(y_te, dt.predict_proba(X_te)[:, 1])
    print(f"{str(d):<12} {tr_auc:<12.4f} {te_auc:<12.4f} {dt.get_n_leaves()}")
```

### 가지치기 파라미터

| 파라미터 | 역할 | 권장값 (금융) |
|---------|------|------------|
| `max_depth` | 최대 트리 깊이 | 3~6 |
| `min_samples_split` | 분기 최소 샘플 수 | 20~50 |
| `min_samples_leaf` | 리프 최소 샘플 수 | 10~30 |
| `min_impurity_decrease` | 불순도 최소 감소량 | 0.001~0.01 |
| `ccp_alpha` | 비용 복잡도 가지치기 | GridSearch 추천 |

## 2. Random Forest

### 원리: Bagging + 랜덤 특성 선택

```
1) Bootstrap 샘플링: 훈련 데이터에서 n개를 복원 추출 → 서로 다른 데이터셋 B개 생성
2) 각 트리 훈련: 매 분기마다 √p개 특성 중 최적을 선택 (p = 전체 특성 수)
3) 앙상블: B개 트리의 예측 평균(회귀) 또는 다수결(분류)
```

**핵심 아이디어**: 상관관계가 낮은 다양한 트리들의 평균 → 분산 감소, 편향 유지

$$\text{Var}(\bar{X}) = \rho \sigma^2 + \frac{1-\rho}{B}\sigma^2 \xrightarrow{B \to \infty} \rho \sigma^2$$

트리 간 상관관계 $\rho$가 작을수록 앙상블 효과가 큽니다.

### 실습: Random Forest 팩터 분석

```python
from sklearn.ensemble import RandomForestClassifier
import matplotlib.pyplot as plt

feature_names = ['모멘텀', 'RSI', '변동성', '거래량변화']

# n_estimators 수렴 분석
oob_errors = []
for n_est in range(10, 301, 10):
    rf = RandomForestClassifier(
        n_estimators=n_est,
        max_depth=5,
        min_samples_leaf=20,
        max_features='sqrt',
        oob_score=True,
        random_state=42,
        n_jobs=-1,
    )
    rf.fit(X_tr, y_tr)
    oob_errors.append(1 - rf.oob_score_)

plt.figure(figsize=(8, 4))
plt.plot(range(10, 301, 10), oob_errors)
plt.xlabel('n_estimators')
plt.ylabel('OOB Error')
plt.title('트리 수 vs OOB Error (수렴 확인)')
plt.axvline(x=100, color='r', linestyle='--', label='일반적 충분값 100')
plt.legend()
plt.tight_layout()
plt.savefig('rf_oob.png', dpi=120)
print("저장: rf_oob.png")

# 최종 모델
rf_final = RandomForestClassifier(
    n_estimators=200,
    max_depth=5,
    min_samples_leaf=20,
    max_features='sqrt',
    oob_score=True,
    random_state=42,
    n_jobs=-1,
)
rf_final.fit(X_tr, y_tr)

te_auc = roc_auc_score(y_te, rf_final.predict_proba(X_te)[:, 1])
print(f"\nRandom Forest 테스트 AUC: {te_auc:.4f}")
print(f"OOB Score: {rf_final.oob_score_:.4f}")
```

### 특성 중요도 (Feature Importance)

```python
# 1) MDI (Mean Decrease in Impurity) — 기본값, 빠르지만 편향 가능
mdi_importance = pd.Series(rf_final.feature_importances_, index=feature_names)

# 2) Permutation Importance — 더 신뢰할 수 있음
from sklearn.inspection import permutation_importance

perm = permutation_importance(rf_final, X_te, y_te, n_repeats=30, random_state=42, n_jobs=-1)
perm_importance = pd.Series(perm.importances_mean, index=feature_names)

fig, axes = plt.subplots(1, 2, figsize=(12, 4))
mdi_importance.sort_values().plot.barh(ax=axes[0], title='MDI Importance')
perm_importance.sort_values().plot.barh(ax=axes[1], title='Permutation Importance (권장)')
plt.tight_layout()
plt.savefig('rf_importance.png', dpi=120)
print("저장: rf_importance.png")

print("\nMDI vs Permutation 비교:")
compare = pd.DataFrame({'MDI': mdi_importance, 'Permutation': perm_importance})
print(compare.sort_values('Permutation', ascending=False))
```

> **주의**: MDI(기본 feature_importances_)는 연속형 특성과 고카디널리티 특성을 과대평가하는 경향이 있습니다.  
> 실제 모델 설명에는 **Permutation Importance** 또는 **SHAP**을 사용하세요.

## 4. 금융 데이터 특화 주의사항

### 팩터 누설 (Look-ahead Bias) 방지

```python
def create_features_no_leakage(prices: pd.Series, window: int = 20) -> pd.DataFrame:
    """미래 데이터를 사용하지 않는 팩터 생성 (shift(1) 필수)"""
    ret = prices.pct_change()

    features = pd.DataFrame(index=prices.index)
    # 모든 특성은 현재 시점 이전 데이터만 사용
    features['mom_5']   = ret.rolling(5).sum().shift(1)
    features['mom_20']  = ret.rolling(20).sum().shift(1)
    features['vol_20']  = ret.rolling(20).std().shift(1)
    features['rsi']     = (ret.clip(lower=0).rolling(14).mean() /
                           ret.abs().rolling(14).mean()).shift(1)
    # 레이블: 다음날 수익률 방향
    features['target']  = (ret.shift(-1) > 0).astype(int)
    return features.dropna()
```

## 핵심 요약

- **의사결정나무**: 해석 용이하지만 과적합 취약 → `max_depth`, `min_samples_leaf`로 제어
- **Random Forest**: 다양한 트리의 앙상블 → 분산 감소 → 더 안정적인 예측
- **OOB Score**: 검증 세트 없이도 성능 추정 가능 (Bootstrap 외 샘플 활용)
- **Feature Importance**: MDI보다 Permutation 또는 SHAP가 더 신뢰할 수 있음
- **시계열에서 `shuffle=False`**: 팩터 생성 시 반드시 `.shift(1)` 적용
