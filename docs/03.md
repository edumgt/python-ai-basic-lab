# 시계열 모델 1: RNN, LSTM, Transformer 기초

> 시계열 모델은 숫자 하나보다 "순서"를 중요하게 봅니다.

---

## 이 문서의 역할

- `03.md`는 시계열 모델의 기본 흐름을 정리하는 문서입니다.
- RNN, LSTM, Transformer가 무엇을 기억하고 무엇을 잘하는지 비교합니다.
- 최신 Transformer 응용은 [04.md](04.md)에서 이어집니다.

## 🌟 초등학생도 이해할 수 있어요

시계열 모델은 **순서가 중요한 이야기**를 읽는 모델입니다.

예를 들어:

- 월요일 비가 왔고
- 화요일도 비가 왔고
- 수요일엔 햇빛이 났다면

목요일 날씨를 맞힐 때는 앞의 흐름이 중요하겠죠.

주가도 비슷합니다.

- 3일 연속 올랐는지
- 갑자기 거래량이 늘었는지
- 지난주부터 분위기가 달라졌는지

를 함께 봐야 합니다.

## 시계열 모델 이름도 말뿌리부터 보면 덜 어렵습니다

| 용어 | 한자/영어 | 말 그대로 뜻 | 초등학생식 이해 |
|------|------|------|------|
| 시계열 | 時系列 / *time series* | 시간의 줄 | 날짜 순서대로 세운 이야기 |
| 순환 | 循環 / *recurrent* | 빙글빙글 다시 돎 | 앞 정보를 다음 칸으로 계속 넘김 |
| 장단기 기억 | 長短期 記憶 / *long short-term memory* | 긴 것도 짧은 것도 기억 | 중요한 건 오래, 덜 중요한 건 짧게 기억 |
| 변환기 | 變換器 / *transformer* | 바꾸는 기계 | 입력을 새 표현으로 바꾸며 중요한 관계 찾기 |
| 주목 | 注目 / *attention* | 눈길 주기 | 중요한 날짜를 콕 집어 보기 |

---

## 왜 시계열 모델이 따로 필요할까요?

주식, 거래량, 예약률, 환율 같은 데이터는 **시간 순서**가 중요합니다.

- 어제와 오늘은 연결되어 있고
- 특정 구간의 충격이 며칠 뒤까지 이어질 수 있고
- 계절성이나 장기 추세가 숨어 있을 수 있습니다

그래서 시계열 모델은
**"숫자 크기"뿐 아니라 "앞뒤 관계"**를 배우려 합니다.

## 시계열에서 자주 나오는 단어 풀이

| 단어 | 쉬운 뜻 | 왜 자주 나오나요? |
|------|------|------|
| 시계열 | 시간 순서대로 줄 세운 데이터 | 주가, 거래량, 환율은 모두 시간 순서가 중요합니다. |
| 시점 | 시간의 한 칸 | 하루, 한 시간, 한 분 같은 한 줄을 뜻합니다. |
| 문맥(context) | 앞뒤 흐름 전체 | 오늘 값만이 아니라 며칠 전 분위기까지 함께 보는 뜻입니다. |
| 의존성 | 앞 시점이 뒤 시점에 주는 영향 | 어제의 급등이 오늘에도 영향을 줄 수 있습니다. |
| 긴 의존성 | 아주 오래전 정보가 지금도 중요한 경우 | 한 달 전 충격이 아직 영향을 주는 상황입니다. |
| 병렬 처리 | 여러 부분을 한꺼번에 계산 | Transformer가 빠르게 계산할 때 자주 나오는 말입니다. |
| 히스토리 길이 | 모델이 뒤를 얼마나 돌아보는지 | 최근 5일, 20일, 60일처럼 입력 길이를 말합니다. |

### 이름에 들어 있는 영어를 그대로 읽어보면

- `RNN`의 `recurrent`는 **되풀이되는**이라는 뜻입니다.
  앞 상태를 다음으로 계속 넘기기 때문에 이런 이름이 붙었습니다.
- `LSTM`의 `memory`는 진짜로 **기억 장치** 같은 역할을 한다는 뜻입니다.
- `Transformer`의 `transform`는 **모양을 바꾸다**는 뜻입니다.
  입력을 그냥 복사하지 않고, 더 알아보기 쉬운 표현으로 바꿔 본다고 생각하면 됩니다.

## 1. RNN

RNN은 순서대로 한 칸씩 읽으면서 이전 상태를 다음 시점으로 넘깁니다.

> 쉬운 비유: 만화책을 1권부터 차례대로 읽으면서 줄거리를 기억하는 친구

| 질문 | 답 |
|------|------|
| 무엇을 잘하나요? | 순서대로 읽는 흐름 이해 |
| 비유하면? | 이야기를 앞에서부터 읽으며 기억하는 사람 |
| 약점은? | 아주 긴 흐름에서는 예전 기억이 흐려지기 쉬움 |

### hidden state는 무엇일까요?

RNN 설명을 보다 보면 `hidden state`라는 말을 자주 봅니다.  
이것은 아주 쉽게 말해 **"지금까지 읽은 내용을 잠깐 적어 두는 메모장"**입니다.

- 어제까지의 분위기를 메모해 두고
- 오늘 데이터를 읽은 뒤
- 메모를 조금 고쳐 다음 날로 넘깁니다

그래서 RNN은 한 칸씩 읽으며 메모를 이어 가는 모델이라고 볼 수 있습니다.

---

## 2. LSTM

LSTM은 RNN의 업그레이드입니다.

핵심은:

- 중요한 정보는 오래 기억하고
- 덜 중요한 정보는 잊도록
- 기억 문을 조절한다는 점입니다

> 쉬운 비유: 시험공부할 때 중요한 공식은 별표 치고 오래 기억하고, 덜 중요한 건 빨리 넘기는 노트 정리

| 질문 | 답 |
|------|------|
| 무엇이 좋아졌나요? | 긴 의존성 처리, 기억 제어 |
| 언제 유용한가요? | 연속 흐름, 추세 지속, 누적 효과 |
| 여전히 어려운 점은? | 아주 긴 구간, 병렬 처리 속도 |

### 게이트(gate)는 왜 필요할까요?

LSTM에는 "잊을지", "기억할지", "새로 넣을지"를 정하는 문 같은 장치가 있습니다.  
이걸 보통 **게이트**라고 부릅니다.

- 중요한 뉴스 충격은 오래 기억하고
- 잠깐 튄 잡음은 빨리 잊고
- 새로 들어온 정보는 적당히 섞는

역할을 한다고 생각하면 됩니다.

## 3. Transformer

Transformer는 순서대로만 읽지 않고,
한 구간 안의 여러 시점을 **한꺼번에 비교**합니다.

핵심은 **Attention**입니다.

> 쉬운 비유: 공책 전체를 펼쳐 놓고 형광펜으로 "여기가 중요!" 하고 표시하는 것

즉,

- "어느 날짜가 지금 예측에 중요한가?"
- "거래량 급증일과 급락일 중 무엇을 더 크게 볼까?"

를 점수처럼 계산합니다.

| 질문 | 답 |
|------|------|
| 무엇을 잘하나요? | 긴 구간 전체 보기, 중요 시점 선택 |
| 비유하면? | 노트를 한 장씩 읽지 않고 펼쳐 놓고 중요 문장을 표시하는 사람 |
| 주의할 점은? | 데이터와 연산량이 큼 |

### Attention은 점수표라고 생각하면 쉽습니다

Transformer는 각 시점이 지금 예측에 얼마나 중요한지  
서로 점수를 매기듯 계산합니다.

예를 들어 오늘 예측을 할 때:

- 어제 거래량 급증은 0.9점
- 일주일 전 급락은 0.7점
- 20일 전 평범한 날은 0.1점

처럼 중요도를 줄 수 있습니다.  
이 점수표가 바로 Attention을 이해하는 좋은 출발점입니다.

---

## 세 모델 비교

| 모델 | 데이터를 보는 방식 | 강점 | 약점 |
|------|------|------|------|
| RNN | 한 칸씩 순서대로 | 구조가 단순 | 긴 기억 약함 |
| LSTM | 순서대로 + 기억 조절 | 긴 패턴에 더 강함 | 느릴 수 있음 |
| Transformer | 전체를 동시에 비교 | 긴 문맥, Attention | 무겁고 데이터 요구 큼 |

### 작은 예시

- 월: +1%
- 화: +1%
- 수: -0.2%
- 목: +0.8%
- 금: 거래량 급증 +2%

이때

- RNN은 "앞에서부터 읽으며 흐름을 기억"
- LSTM은 "중요한 상승 흐름을 더 오래 기억"
- Transformer는 "금요일 거래량 급증이 제일 중요했네!" 하고 더 크게 볼 수 있습니다.

## 공부할 때 이렇게 비교하세요

1. RNN은 "순서대로 읽는 모델"이라고 기억합니다.
2. LSTM은 "오래 기억하도록 개선한 모델"이라고 정리합니다.
3. Transformer는 "전체를 같이 보고 중요한 부분을 고르는 모델"이라고 정리합니다.
4. 웹앱에서는 정확도보다도 **어떤 입력과 어떤 출력이 연결되는지**를 먼저 봅니다.

## 한 번 더 쉬운 말로 정리

- RNN: 한 줄씩 읽는 모델
- LSTM: 중요한 건 오래 기억하는 모델
- Transformer: 전체를 펼쳐 놓고 중요한 부분에 집중하는 모델
