# 웹앱 실습 3: 개념 놀이터로 지도학습과 군집화 익히기

> 버튼을 누르며 지도학습, 비지도학습, 레이블, 군집화를 한 화면에서 비교합니다.

---

## 이 실습의 목표

- 모델 이름을 외우기 전에 데이터 개념을 먼저 눈에 익힙니다.
- `상승/하락 정답이 있는 문제`와 `비슷한 점끼리 묶는 문제`의 차이를 분명히 봅니다.
- `지도학습`, `비지도학습`, `레이블` 뜻이 헷갈리면 [voca.md](voca.md)를 같이 봅니다.

---

이 실습은

- "정답을 보고 공부하는 문제"
- "정답 없이 비슷한 것끼리 모으는 문제"

를 한 화면에서 비교하는 놀이터입니다.

색연필을 색깔별로 정리하는 것처럼 보면 훨씬 쉽습니다.

---

개념 비교는 `주식 AI 실험실` 안의 작은 개념 실습 영역에서 진행합니다.

## 꼭 눌러볼 모드

| 모드 | 무엇을 배우나 |
|------|------|
| `지도학습` | 정답 레이블이 있는 문제 |
| `비지도학습` | 정답 없이 구조 찾기 |
| `레이블` | 정답 열이 왜 중요한지 |
| `군집화` | K-Means가 비슷한 점을 어떻게 묶는지 |

## 개념 놀이터 단어 사전

| 단어 | 쉬운 뜻 | 이 화면에서 어떻게 보이나 |
|------|------|------|
| 지도학습 | 정답을 보며 배우는 방식 | `상승/하락` 같은 정답 열이 함께 보입니다. |
| 비지도학습 | 정답 없이 구조를 찾는 방식 | 점들의 모양과 거리만 보고 묶습니다. |
| 레이블 | 정답 이름표 | 마지막 열에 0/1 같은 값으로 붙는 경우가 많습니다. |
| 군집화 | 비슷한 점끼리 무리 만들기 | 비슷한 색 점끼리 그룹이 생깁니다. |
| K값 | 만들 무리 개수 | 2면 두 무리, 3이면 세 무리입니다. |
| 산점도 | 점으로 데이터를 보여 주는 그래프 | 점이 얼마나 모여 있는지 한눈에 볼 수 있습니다. |

---

## 실습 순서

1. `지도학습` 모드를 눌러 표와 점 색을 봅니다.
2. `비지도학습` 모드로 바꿔 정답이 숨겨졌을 때 화면이 어떻게 달라지는지 봅니다.
3. `레이블` 모드에서 마지막 열이 왜 중요한지 확인합니다.
4. `군집화` 모드로 가서 `K` 값을 2, 3, 4로 바꿔봅니다.
5. 같은 데이터라도 묶이는 방식이 달라지는지 확인합니다.

### 레이블은 왜 중요할까요?

레이블은 데이터에 붙은 **정답 이름표**입니다.

예를 들어:

- 오늘 데이터를 보고
- 내일 상승하면 `1`
- 내일 하락하면 `0`

처럼 붙일 수 있습니다.  
지도학습 모델은 이 이름표를 보면서  
"어떤 입력이면 1이 되는지"를 배우게 됩니다.

반대로 군집화는 이런 이름표 없이도  
점들끼리 비슷한 모양을 보고 스스로 묶습니다.

## 화면에서 볼 포인트

| 위치 | 확인 포인트 |
|------|------|
| 상단 모드 버튼 | 학습 방식 전환 |
| 표 영역 | 레이블 유무 차이 |
| 산점도 | 점 색과 그룹 변화 |
| K 슬라이더 | 군집 수 조절 |

---

## 관찰 질문

- 정답 레이블이 보일 때와 숨겨질 때 느낌이 어떻게 다른가요?
- `K=2`와 `K=4`에서 어떤 점이 달라지나요?
- 군집화 결과를 "정답"처럼 보지 말아야 하는 이유는 무엇인가요?

---

## 작은 과제

- `지도학습`과 `비지도학습`의 차이를 2문장으로 적습니다.
- `군집화` 모드에서 가장 자연스럽다고 느낀 `K`를 하나 고릅니다.
- 그 이유를 "점이 몇 개 무리로 보이는지" 기준으로 적습니다.
