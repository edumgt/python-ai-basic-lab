# 웹앱 실습 2: 같은 데이터로 모델 4종 비교하기

> 같은 삼성전자 샘플로 로지스틱 회귀, 랜덤 포레스트, 그래디언트 부스팅, 신경망을 비교합니다.

---

## 이 실습의 목표

- 데이터는 같게 두고 모델만 바꾸면 무엇이 달라지는지 봅니다.
- "모델 선택"이 결과 해석에 어떤 영향을 주는지 감을 잡습니다.
- 어려운 모델 이름은 [voca.md](voca.md)의 한자·영어 풀이와 함께 읽습니다.

---

이 실습은 같은 문제를

- 친구 A
- 친구 B
- 친구 C
- 친구 D

에게 풀게 해 보고,  
누가 어떤 방식으로 잘 푸는지 비교하는 것과 비슷합니다.

---

## 열 페이지

- [주식 AI 실험실](/lab)

### 먼저 누를 메뉴

- 왼쪽 데이터 입력의 `삼성전자`
- 모델 버튼 `로지스틱 회귀`
- 모델 버튼 `랜덤 포레스트`
- 모델 버튼 `그래디언트 부스팅`
- 모델 버튼 `신경망`

---

## 실습 순서

1. `삼성전자` 샘플을 불러옵니다.
2. 모델 버튼 `로지스틱 회귀`를 누르고 실행합니다.
3. `정확도`, `AUC`, `상승 확률`, `특성 중요도`를 기록합니다.
4. 같은 데이터로 `랜덤 포레스트`, `그래디언트 부스팅`, `신경망` 버튼을 차례대로 눌러 실행합니다.
5. 어느 모델이 가장 안정적으로 보이는지 비교합니다.

## 비교할 때 자주 보는 단어 뜻

| 단어 | 쉬운 뜻 | 비교할 때 포인트 |
|------|------|------|
| 기준선(baseline) | 가장 먼저 놓고 비교하는 기본 모델 | 로지스틱 회귀가 자주 기준선이 됩니다. |
| 확률(probability) | 오를 가능성을 0~1이나 %로 나타낸 값 | 0.8이면 꽤 자신 있게 오른다고 본다는 뜻입니다. |
| 중요도(importance) | 모델이 어떤 특성을 많이 참고했는지 | 이유를 읽을 때 중요합니다. |
| 안정적 | 한 번만 좋지 않고 여러 번 비슷하게 괜찮음 | 실전에선 최고점보다 안정성이 더 중요할 수 있습니다. |
| 복잡도 | 모델 구조가 얼마나 복잡한지 | 복잡한 모델은 성능이 좋을 수 있지만 설명이 어려워질 수 있습니다. |

## 해석 힌트

- 로지스틱 회귀는 해석이 쉬운 기준선입니다.
- 랜덤 포레스트는 중요 특성 읽기가 편합니다.
- 그래디언트 부스팅은 보통 강한 기준선 역할을 합니다.
- 신경망은 복잡한 조합을 배울 수 있지만 해석은 상대적으로 어렵습니다.

### 예시로 읽어보면

- 로지스틱 회귀 `AUC 0.62`는 기본선으로는 괜찮을 수 있습니다.
- 랜덤 포레스트 `AUC 0.71`이라면 구분 능력이 더 좋아졌다고 볼 수 있습니다.
- 그런데 신경망이 `AUC 0.72`라도 중요 특성을 거의 설명하기 어렵다면, 초반 실험에서는 랜덤 포레스트가 더 편할 수 있습니다.

즉, "가장 높은 숫자"와 "가장 쓰기 쉬운 모델"이 꼭 같지는 않습니다.

---

## 작은 과제

- 네 모델 중 `AUC`가 가장 높은 모델을 찾습니다.
- 네 모델 중 중요 특성 순위가 가장 다르게 나온 모델을 찾습니다.
- "실전에서 먼저 돌려볼 모델" 하나와 그 이유를 적습니다.
