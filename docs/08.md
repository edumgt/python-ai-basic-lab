# 웹앱 실습 4: 뉴런 계산과 신경망 맛보기

> 숫자를 바꿔가며 가중합, 편향, 시그모이드, 오차, 비용을 직접 확인합니다.

---

## 이 실습의 목표

- 신경망을 "어려운 블랙박스"가 아니라 작은 계산의 반복으로 이해합니다.
- MLP 실습으로 넘어가기 전에 뉴런 1개의 계산을 손에 익힙니다.
- `뉴런`, `가중치`, `편향`, `시그모이드` 뜻이 낯설면 [voca.md](voca.md)를 먼저 봅니다.

---

신경망도 사실은 갑자기 마법처럼 답을 내는 게 아니라,

- 숫자를 곱하고
- 더하고
- 살짝 조정하고
- 마지막에 확률처럼 바꾸는

작은 계산을 여러 번 하는 것입니다.

즉, 큰 로봇도 알고 보면 **작은 계산 블록**이 많이 모인 것이라고 생각하면 됩니다.

---

## 먼저 볼 개념

| 항목 | 쉬운 뜻 |
|------|------|
| 가중합 | 입력마다 중요도를 곱해 더한 값 |
| 편향 | 기본 점수 조정값 |
| 시그모이드 | 결과를 0~1 사이 확률처럼 바꾸는 함수 |
| 오차 | 예측과 정답의 차이 |
| 비용 | 오차를 한 숫자로 요약한 값 |

## 신경망 단어를 더 쉬운 말로

| 단어 | 쉬운 뜻 | 비유 |
|------|------|------|
| 입력 | 모델에 넣는 재료 | 문제를 풀기 전에 보는 힌트 |
| 가중치 | 각 힌트의 중요도 | 어떤 힌트에 형광펜을 더 진하게 칠한 정도 |
| 편향 | 기본 시작점 | 경기를 시작하기 전 기본 점수 |
| 활성화 함수 | 계산 결과를 다음 단계로 넘기는 변환 규칙 | 너무 큰 값을 읽기 쉬운 모양으로 바꾸는 장치 |
| 시그모이드 | 0~1 사이로 눌러 주는 활성화 함수 | 확률처럼 읽기 쉽게 만드는 스위치 |
| 손실(loss) / 비용(cost) | 틀린 정도를 숫자로 모은 값 | 많이 틀릴수록 커지는 벌점 |

---

## 실습 순서

1. `가중합` 모드에서 입력과 가중치를 바꿉니다.
2. `편향` 모드에서 기준점이 어떻게 움직이는지 봅니다.
3. `시그모이드` 모드에서 값이 0~1로 압축되는 것을 확인합니다.
4. `오차`, `비용` 모드에서 정답과 예측 차이가 어떻게 반영되는지 봅니다.
5. 그 다음 같은 화면에서 모델 버튼 `신경망`을 눌러 실제 주가 입력에 적용합니다.

### 아주 작은 계산 예시

입력이 2개 있다고 해봅시다.

- 가격 변화율 = 0.6
- 거래량 변화율 = 0.2

가중치가 각각

- 0.8
- 0.3

이라면, 모델은

- `0.6 x 0.8`
- `0.2 x 0.3`

을 계산해 더한 뒤, 편향을 더하고, 마지막에 시그모이드로 0~1 사이 값으로 바꿀 수 있습니다.

이렇게 보면 신경망도 결국  
**중요도를 곱해 점수를 만들고, 그 점수를 읽기 쉽게 바꾸는 계산**이라는 걸 알 수 있습니다.

## 관찰 질문

- 가중치가 커지면 어떤 입력이 더 중요해지나요?
- 편향은 왜 "기본 점수"라고 부를 수 있나요?
- 시그모이드가 있으면 왜 결과를 확률처럼 읽기 쉬워지나요?

---

## 작은 과제

- 입력 하나의 가중치를 크게 바꾸고 결과 변화를 적습니다.
- 편향을 음수와 양수로 바꾸며 출력이 어떻게 달라지는지 적습니다.
- 마지막에 `NN` 모델 화면으로 넘어가서 가장 중요한 특성 1개를 기록합니다.
