# 웹앱 실습 1: 학습 허브에서 한 챕터 실행하기

> 이 저장소의 기본 사용법을 익히는 실습입니다.

---

## 이 실습의 목표

- 메인 학습 허브(`/`)에서 문서, 소스, 실행 결과, 웹앱 가이드를 오가는 흐름을 익힙니다.
- "문서를 읽고 끝"이 아니라 "챕터를 실행하고 결과를 읽는 습관"을 만듭니다.
- 낯선 단어가 나오면 [voca.md](voca.md)를 바로 펼쳐 보며 실습합니다.

---

이 화면은 게임으로 치면 **메인 로비**입니다.

- 왼쪽에서는 고를 수 있고
- 가운데서는 설명을 읽을 수 있고
- 실행 버튼을 누르면 결과가 바로 나옵니다

즉, "무엇을 배울지 고르고 바로 해보는 교실"이라고 생각하면 쉽습니다.

## 추천 실습 메뉴

처음에는 아래 셋 중 하나를 누르면 좋습니다.

| 메뉴/버튼 | 추천 이유 |
|------|------|
| `로지스틱 회귀` | 가장 쉬운 분류 기준선 |
| `랜덤 포레스트` | 앙상블과 중요도 확인 |
| `신경망` | 딥러닝 입문 |

## 화면 단어 사전

| 화면 단어 | 쉬운 뜻 | 이 실습에서 하는 일 |
|------|------|------|
| 메인 학습 허브 | 학습 자료가 모여 있는 첫 화면 | 어디로 이동할지 정하는 출발점입니다. |
| 메뉴 | 위쪽 이동 버튼 | `실험실`, `예측`, `데이터셋`, `호텔주가`를 옮겨 다닙니다. |
| 샘플 데이터 | 미리 준비된 연습용 데이터 | 직접 파일을 만들지 않아도 바로 실습할 수 있습니다. |
| 모델 버튼 | 어떤 AI를 쓸지 고르는 버튼 | 같은 데이터로 모델만 바꿔 볼 수 있습니다. |
| 결과 카드 | 핵심 숫자를 크게 보여 주는 칸 | accuracy, AUC 같은 값을 빠르게 읽습니다. |
| 차트 | 흐름을 그림으로 보여 주는 영역 | 숫자만 봤을 때 놓친 변화를 확인합니다. |

## 실습 순서

1. [메인 학습 허브](/) 상단의 `🔬 실험실` 메뉴를 눌러 [주식 AI 실험실](/lab)로 이동합니다.
2. 왼쪽 데이터 입력에서 `삼성전자` 같은 샘플을 불러옵니다.
3. 아래 `예측 모델 선택`에서 `로지스틱 회귀` 또는 `랜덤 포레스트`를 고릅니다.
4. `AI 분석 시작!` 버튼을 눌러 결과를 생성합니다.
5. 성능 카드와 차트에서 숫자를 읽고, 필요한 경우 다시 다른 모델 버튼을 눌러 비교합니다.
6. 같은 데이터로 모델만 바꿔 보며 어떤 값이 달라지는지 확인합니다.

### `AI 분석 시작!` 버튼을 누르면 무슨 일이 일어날까요?

아주 단순하게 보면 아래 순서로 진행됩니다.

1. 화면에 있는 데이터를 읽습니다.
2. 선택한 모델로 학습과 예측을 합니다.
3. 성능 숫자를 계산합니다.
4. 중요한 특성과 차트를 보여 줍니다.

즉, 버튼 하나가  
`데이터 읽기 → 모델 실행 → 결과 정리`를 묶어서 해준다고 생각하면 됩니다.

## 화면에서 꼭 볼 것

| 위치 | 확인 포인트 |
|------|------|
| 상단 메뉴 | `실험실`, `예측`, `데이터셋`, `호텔주가` 이동 흐름 |
| 왼쪽 데이터 입력 | 샘플 불러오기와 CSV 입력 |
| 모델 버튼 영역 | 모델별 차이를 바로 비교 |
| `AI 분석 시작!` 버튼 | 현재 선택한 모델 실행 |
| 오른쪽 결과 영역 | 정확도, 확률, 중요도, 차트 |

## 관찰 질문

- 같은 데이터를 두고 모델 버튼만 바꿔 볼 때 이해가 더 쉬워지나요?
- 실행 결과에서 어떤 숫자가 가장 먼저 눈에 들어오나요?
- 상단 메뉴가 "다음 실습 화면"을 정하는 데 도움이 되나요?
