LoRA 파라미터 압축 시뮬레이터

chapter114 · numpy 기반 저랭크 행렬 분해 시뮬레이션

핵심 수식
ΔW = B × A
B: (d × r) ← 0으로 초기화
A: (r × k) ← 랜덤 초기화
r ≪ min(d, k)

W₀ + BA로 출력 = 원본 + 적응

동결 vs 학습
W₀ (동결) — 업데이트 안 함
B, A (학습) — 파라미터 1% 미만

추론 시 W₀ + BA로 병합 가능

랭크(r) 선택 가이드
r = 4빠름, 최소 파라미터
r = 8균형점 (권장)
r = 16표현력↑, 파라미터↑
r = 32Full에 가까워짐
설정 Full 파라미터 LoRA 파라미터 압축률 절감 수

LLM 파인튜닝 방법 비교

방법 업데이트 파라미터 메모리 속도 특징
Full Fine-Tuning 전체 매우 큼 느림 성능 최대, 비용 최대
LoRA B, A 행렬만 작음 빠름 원본 동결, 저랭크 행렬 추가. 파라미터 1% 미만
QLoRA B, A + 양자화 매우 작음 빠름 4비트 NF4 양자화. 소비자 GPU(6GB)에서 7B 가능
Instruction Tuning 선택 가능 가변 가변 지시 따르기 특화 데이터로 학습
Prompt Tuning 소프트 프롬프트만 최소 최빠름 학습 가능한 토큰 추가. 모델 자체는 동결

QLoRA 메모리 절감 효과 (LLaMA 7B 기준)

~112 GB
Full Fine-Tuning (bf16)
~14 GB
LoRA (bf16)
~6 GB
QLoRA (4bit + LoRA) ← RTX 3080
